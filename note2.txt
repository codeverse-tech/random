## 2️⃣ **Developer Documentation / Quickstart**

### **REST API Endpoints (MVP)**

| Endpoint           | Method | Description                    |
| ------------------ | ------ | ------------------------------ |
| `/auth/otp/send`   | POST   | Sends OTP via WhatsApp/SMS     |
| `/auth/otp/verify` | POST   | Verifies OTP and returns token |
| `/auth/login`      | POST   | Phone/email login              |
| `/auth/refresh`    | POST   | Refresh JWT token              |
| `/auth/logout`     | POST   | Revoke session                 |

**Example: Phone login with JS SDK**

```js
import credensync from 'credensync-sdk';

const response = await credensync.loginWithPhone("+2348012345678");
console.log(response.status); // "otp_sent"

await credensync.verifyOTP("+2348012345678", "48291");
```

## 8️⃣ **Founder Story / Vision**

CredenSync was born from a simple observation:
**Africa’s digital identity is broken.**
Email is unreliable. SMS is flaky. WhatsApp dominates communication.

As a solo founder and backend engineer, I realized the solution had to be:

* **Phone-first, WhatsApp-first**
* **Plug-and-play for indie devs and startups**
* **Secure and reliable by default**

CredenSync is about removing friction, enabling developers to focus on their product, and building a **local-first identity solution** for Africa.
